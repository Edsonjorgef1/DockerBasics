# 1 - $ docker run -d -p 80:80 docker/getting-started # docker run -dp 80:80 docker/getting-started # -> Explanation: # # -d -> run the container in detached mode (in the background) # # -p 80:80 -> map port 80 of the host to port 80 in the container # # docker/getting-started -> the image to use # 2 - $ docker build -t getting-started . # -> This command used the Dockerfile to build a new container image. Since we named the image "getting-started", # we can refer to that image when we run a container. The . at the end of the docker build command tells that # Docker should look for the Dockerfile in the current directory.
